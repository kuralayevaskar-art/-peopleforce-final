# HR PeopleOps Platform

Модульный монолит для HR-задач и базовых PeopleOps-процессов: сотрудники, отделы, должности, документы, структура компании, посещаемость и интеграции с AD/ZKTeco/Synology.

## Главная цель

Платформа должна стать внутренней HR-системой, через которую администратор сможет создавать сотрудников, заводить им учетные записи в Active Directory, назначать доступы по отделам, хранить документы и фото, а также получать посещаемость из ZKTeco.

## Основные модули

- Сотрудники: список, карточка сотрудника, фото, документы, должность, отдел, руководитель.
- Отделы и должности: структура компании, группировка сотрудников, оргструктура.
- Active Directory: создание пользователей и синхронизация данных.
- ZKTeco: получение посещаемости с платформы `10.1.70.2`.
- Synology: хранение папок и файлов сотрудников в `Z:\Global\people`.
- Роли и доступы: администратор видит настройки, обычный пользователь не видит.
- Self-service сайт: новый пользователь сам заполняет свои данные перед созданием аккаунта.

## Active Directory

AD сервер: `10.1.10.11`.

Через платформу HR/admin заполняет ФИО, отдел и личную почту нового сотрудника. Система должна:

1. Сгенерировать логин.
2. Сгенерировать корпоративную почту.
3. Сгенерировать временный пароль по стандарту Windows/AD.
4. Создать пользователя в Active Directory.
5. Назначить доступы по отделу.
6. Отправить логин, почту и временный пароль на личную почту сотрудника.
7. Создать карточку сотрудника в HR-платформе.
8. Создать папки сотрудника на Synology.

Пример:

- ФИО: `Куралаев Аскар Акатаевич`
- Логин: `a.kuralayev`
- Почта: `a.kuralayev@dmuk.edu.kz`

## ZKTeco

Посещаемость будет браться из платформы ZKTeco:

- Host: `10.1.70.2`
- Назначение: импорт посещаемости сотрудников в HR-платформу.
- Логин и пароль должны храниться только в локальном secret-файле.

## Synology документы

Synology:

- Host: `10.1.30.49`
- Путь: `Z:\Global\people`

Для каждого сотрудника создается папка по логину.

Пример:

```text
Z:\Global\people\a.kuralayev
Z:\Global\people\a.kuralayev\agreement
Z:\Global\people\a.kuralayev\photo
Z:\Global\people\a.kuralayev\documents
Z:\Global\people\a.kuralayev\other
```

Через карточку сотрудника платформа должна давать доступ к загрузке фото, договора, документов и других файлов.

## Где писать логины и пароли

Не писать пароли в код и README.

Создать файл:

```text
backend/src/main/resources/application-secrets.properties
```

Шаблон находится здесь:

```text
docs/secrets-template.md
```

Туда позже вручную вписать:

- AD bind user / password
- ZKTeco username / password
- Synology username / password
- Mail SMTP username / password

## Роли

- `SUPER_ADMIN`: полный доступ.
- `HR_ADMIN`: сотрудники, документы, отделы, должности.
- `INTEGRATION_ADMIN`: настройки AD, ZKTeco, Synology.
- `MANAGER`: свой отдел и подчиненные.
- `EMPLOYEE`: только своя карточка и разрешенные данные.

Обычные пользователи не должны видеть меню `Настройки`.
